## 项目内容及要求

本项目是一个基于 **Vue3 + SpringBoot + MySQL + Redis** 技术栈的校园二手物品交易平台综合实训，旨在实现一个功能完善、界面友好、安全可靠的校园C2C二手交易Web应用。详细要求如下：

### 一、系统总体要求

- 采用前后端分离架构，前端使用Vue3框架，后端使用SpringBoot框架，数据库使用MySQL，缓存使用Redis。
- 实现完整的用户认证与授权机制（JWT + SpringSecurity）。
- 系统需支持图片上传、富文本编辑、实时消息通知等交互功能。
- 界面要求响应式设计，兼容PC端主流浏览器。

### 二、功能模块要求

- **用户管理模块**
    - 用户注册（邮箱/手机号验证）、登录、密码找回
    - 个人中心：头像上传、个人信息编辑、收货地址管理
    - 用户信誉评分与实名认证（可选）

- **商品管理模块**
    - 商品发布：支持多图上传、分类选择、价格设定、商品描述（富文本）
    - 商品浏览：首页推荐、分类筛选、关键词搜索、排序（价格/时间/热度）
    - 商品详情：图片轮播、卖家信息、收藏功能、浏览量统计
    - 商品状态管理：上架/下架/已售出/删除

- **订单交易模块**
    - 购物车功能（可选）或直接购买
    - 订单生成、订单状态流转（待付款/待发货/待收货/已完成/已取消）
    - 在线支付接口模拟（或集成第三方支付沙箱环境）
    - 物流信息录入与查询（模拟）

- **消息与沟通模块**
    - 站内信/留言板：买家与卖家沟通
    - 系统通知：订单状态变更、审核结果通知
    - 消息已读/未读标记

- **后台管理模块**
    - 用户管理：用户列表、禁用/启用、权限分配
    - 商品审核：待审核商品列表、通过/驳回、违规下架
    - 订单管理：订单查询、异常处理
    - 数据统计：注册用户量、商品发布量、成交量、分类占比图表
    - 公告管理：系统公告发布与展示

- **（可选）扩展功能**
    - 基于用户行为的商品推荐算法
    - 敏感词过滤与内容审核
    - 交易纠纷举报与处理

## 主要开发设计流程及技术路线

1. **技术调研与准备**
   熟悉Vue3组合式API、SpringBoot核心特性、MySQL数据库设计、Redis缓存机制；查阅RESTful API设计规范、前后端分离项目最佳实践相关文献。

2. **需求分析**
   进行系统角色分析（游客、普通用户、卖家、管理员）；绘制用例图；完成功能性与非功能性需求分析文档。

3. **系统架构设计**
   绘制系统架构图、ER图、时序图及各功能模块流程图；确定前后端接口规范（RESTful API）；设计数据库表结构及索引策略。

4. **数据库与缓存设计**
   完成MySQL数据库表设计（用户表、商品表、订单表、消息表、分类表等）；设计Redis缓存策略（热点商品、用户会话、验证码等）。

5. **开发环境搭建**
   搭建前端Vue3项目脚手架；搭建SpringBoot后端项目；配置MySQL与Redis；配置跨域与统一返回结果封装。

6. **后端核心功能开发**
   实现用户认证（JWT + SpringSecurity）、商品CRUD、订单业务逻辑、文件上传（OSS/本地）、消息通知等接口。

7. **前端界面开发**
   实现登录/注册页、首页、商品列表/详情页、个人中心、卖家后台、管理后台等页面；实现前后端数据交互与状态管理。

8. **系统联调与测试**
   前后端接口联调；进行单元测试、集成测试；使用Postman进行接口测试；进行压力测试与性能优化。

9. **文档编写与答辩准备**
   编写需求分析文档、系统设计文档、数据库设计文档、接口文档、测试报告、用户使用说明书；准备答辩PPT。

## 项目重难点

1. **前后端分离架构的理解与接口设计**：需要深入理解RESTful API设计规范，掌握前后端数据交互模式（Axios拦截器、统一错误处理、跨域配置），以及JWT无状态认证机制的实现。

2. **数据库表设计与业务逻辑复杂性**：二手交易平台涉及用户、商品、订单、消息等多表关联，需要合理设计表结构、外键约束、事务处理（如订单创建与库存扣减的原子性），以及复杂查询的SQL优化。

3. **文件上传与图片管理**：多图片上传、缩略图生成、图片存储方案（本地/云存储）的选择与实现。

4. **并发与性能优化**：高并发场景下的数据库锁机制、Redis缓存穿透/击穿/雪崩的防护策略、热点商品数据的缓存设计。

5. **安全机制实现**：用户密码加密存储（BCrypt）、JWT令牌刷新机制、接口防刷、SQL注入与XSS攻击防护。

## 学生工作分工及考核内容（含评分标准）

选定组长，由一人总体负责进行小组分工，系统实现由小组成员共同完成。

**推荐分工如下：**

1. **共同完成**：需求分析、系统架构设计、数据库设计、接口规范制定。（全组成员）

2. **后端开发（2人）**：负责SpringBoot后端核心功能实现，包括用户认证、商品管理、订单业务、文件上传、Redis缓存、数据库设计与优化。

3. **前端开发（2人）**：负责Vue3前端界面实现，包括页面设计与开发、组件封装、状态管理、前后端联调、响应式适配。

4. **测试与文档（全组成员）**：负责系统功能测试、接口测试（Postman）、Bug记录与跟踪；同时负责管理后台前端开发、数据统计图表实现、文档整合。

项目开发的各阶段文档由小组成员共同负责。
