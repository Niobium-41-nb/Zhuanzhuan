## 1. 引言 (Introduction)

### 1.1 目的

本软件需求规格说明书（SRS）的目的是全面、系统地描述"校园二手物品交易平台"的功能需求、非功能需求、外部接口需求和约束条件，为后续的系统设计、编码实现、测试验收提供统一的依据和基准。

### 1.2 范围

本项目是一个基于 **Vue3 + SpringBoot + MySQL + Redis** 技术栈的校园二手物品交易平台综合实训项目，旨在实现一个功能完善、界面友好、安全可靠的校园 C2C 二手交易 Web 应用。

本系统覆盖以下功能域：
- 用户管理
- 商品管理
- 订单交易
- 消息与沟通
- 后台管理
- 系统安全与审计

## 2. 总体描述 (General Description)

### 2.1 产品视角

本系统是一个独立的校园二手物品交易 Web 应用平台，采用前后端分离架构。系统不依赖其他外部系统运行，但通过与数据库（MySQL）和缓存（Redis）的交互实现数据持久化与高性能访问。

**系统上下文：**
- **前端：** Vue3 单页应用（SPA），运行于用户浏览器
- **后端：** SpringBoot RESTful API 服务
- **数据库：** MySQL 关系型数据库
- **缓存层：** Redis 缓存服务
- **文件存储：** 本地存储或云 OSS

### 2.3 用户特征

| 用户角色 | 描述 | 技术能力 | 使用频率 |
|:---|:---|:---|:---|
| **游客** | 未注册的浏览者，可浏览商品但不能进行交易操作 | 低 | 偶尔 |
| **普通用户/买家** | 已注册用户，可浏览、购买商品、管理个人信息 | 中等 | 经常 |
| **卖家** | 已注册并通过认证的用户，可发布和管理商品 | 中等 | 频繁 |
| **管理员** | 系统运营人员，拥有后台管理权限 | 高 | 每日 |

### 2.4 约束条件

| 约束项 | 内容 |
|:---|:---|
| 技术栈约束 | 前端必须使用 Vue3，后端必须使用 SpringBoot |
| 数据库约束 | 必须使用 MySQL 关系型数据库 |
| 缓存约束 | 必须使用 Redis 作为缓存层 |
| 认证约束 | 必须使用 JWT + SpringSecurity 实现认证授权 |
| 架构约束 | 必须采用前后端分离架构 |
| 兼容性约束 | 必须兼容 PC 端主流浏览器（Chrome、Firefox、Edge、Safari） |
| 开发周期约束 | 需在实训周前完成核心功能编码与联调 |

### 2.5 假设与依赖

- 假设目标用户具备基本的 Web 浏览操作能力
- 假设校园网络环境能够正常访问 Web 应用
- 依赖 MySQL 数据库提供可靠的数据持久化服务
- 依赖 Redis 提供高性能缓存服务
- 依赖邮件/短信服务提供验证码发送功能（或使用模拟实现）

## 3. 功能需求 (Functional Requirements)

### 3.1 用户管理模块

#### 3.1.1 用户注册

| 需求标识 | FR-UM-01 |
|:---|:---|
| **名称** | 用户注册 |
| **描述** | 系统应允许新用户通过邮箱或手机号进行注册 |
| **前置条件** | 用户未登录，输入信息合法 |
| **后置条件** | 系统创建新用户账户，返回注册成功信息 |
| **输入** | 用户名、邮箱/手机号、密码（加密传输） |
| **处理逻辑** | 1. 验证邮箱/手机号唯一性<br>2. 发送验证码并校验<br>3. 密码使用 BCrypt 加密存储<br>4. 创建用户记录并返回 Token |
| **异常处理** | 邮箱/手机号已注册时提示用户；验证码错误时重试 |

#### 3.1.2 用户登录

| 需求标识 | FR-UM-02 |
|:---|:---|
| **名称** | 用户登录 |
| **描述** | 系统应支持用户通过邮箱/手机号 + 密码或验证码登录 |
| **前置条件** | 用户已注册 |
| **后置条件** | 系统生成 JWT Token，用户进入已登录状态 |
| **处理逻辑** | 1. 校验用户凭证<br>2. 验证密码（BCrypt）<br>3. 生成 JWT Token（含用户角色信息）<br>4. 返回 Token 至前端 |
| **异常处理** | 密码错误超过 5 次时临时锁定账户 15 分钟 |

#### 3.1.3 密码找回

| 需求标识 | FR-UM-03 |
|:---|:---|
| **名称** | 密码找回 |
| **描述** | 系统应支持用户通过绑定的邮箱/手机号重置密码 |
| **处理逻辑** | 1. 用户提交邮箱/手机号<br>2. 发送重置链接/验证码<br>3. 验证通过后允许设置新密码<br>4. 新密码加密存储并清除旧 Token |

#### 3.1.4 个人信息管理

| 需求标识 | FR-UM-04 |
|:---|:---|
| **名称** | 个人信息管理 |
| **描述** | 已登录用户可在个人中心查看和编辑个人信息 |
| **功能项** | 头像上传（支持 jpg/png，限制 2MB）、昵称修改、个人简介编辑、绑定邮箱/手机号更换 |
| **前置条件** | 用户已登录（JWT 校验通过） |

#### 3.1.5 收货地址管理

| 需求标识 | FR-UM-05 |
|:---|:---|
| **名称** | 收货地址管理 |
| **描述** | 用户可添加、编辑、删除、设置默认收货地址 |
| **地址字段** | 收件人、联系电话、省份/城市/区县、详细地址、邮政编码 |

#### 3.1.6 用户信誉评分

| 需求标识 | FR-UM-06 |
|:---|:---|
| **名称** | 用户信誉评分 |
| **描述** | 交易完成后买卖双方可互相评分，系统汇总显示用户信誉等级 |
| **评分范围** | 1-5 星 |
| **显示** | 用户主页展示平均评分和评价总数 |

### 3.2 商品管理模块

#### 3.2.1 商品发布

| 需求标识 | FR-GM-01 |
|:---|:---|
| **名称** | 商品发布 |
| **描述** | 卖家可发布新商品信息 |
| **前置条件** | 用户已登录且通过卖家认证 |
| **输入项** | 商品标题、描述（富文本）、分类、价格、成色、交易方式、图片（最多 9 张） |
| **处理逻辑** | 1. 图片上传至 OSS/本地存储<br>2. 商品信息写入数据库（状态为"待审核"）<br>3. 发送审核通知至管理员 |
| **约束** | 标题长度 5-50 字；价格精确到分；图片单张不超过 5MB |

#### 3.2.2 商品浏览与搜索

| 需求标识 | FR-GM-02 |
|:---|:---|
| **名称** | 商品浏览与搜索 |
| **描述** | 用户可浏览商品列表，支持分类筛选、关键词搜索和多维度排序 |
| **搜索方式** | 关键词搜索（商品标题/描述）、分类筛选、价格区间筛选、成色筛选 |
| **排序方式** | 按发布时间、价格（升/降序）、热度（浏览量/收藏量） |
| **分页** | 支持分页加载，每页默认 20 条 |

#### 3.2.3 商品详情

| 需求标识 | FR-GM-03 |
|:---|:---|
| **名称** | 商品详情 |
| **描述** | 用户可查看商品的完整信息 |
| **展示内容** | 图片轮播、商品标题、价格、成色、描述、卖家信息、发布时间、浏览量、收藏数 |
| **附加功能** | 收藏/取消收藏商品、举报商品 |

#### 3.2.4 商品状态管理

| 需求标识 | FR-GM-04 |
|:---|:---|
| **名称** | 商品状态管理 |
| **描述** | 卖家可管理自己发布的商品状态 |
| **状态流转** | 待审核 → 在售（审核通过）→ 已下架 / 已售出 / 审核驳回 |

## 4. 外部接口需求 (External Interface Requirements)

### 4.1 用户接口

| 接口项 | 描述 |
|:---|:---|
| 前端框架 | Vue3 + Element Plus / Vuetify UI 组件库 |
| 页面类型 | 响应式 Web 页面，兼容 PC 端主流浏览器 |

### 4.2 硬件接口

无特殊硬件接口要求，系统运行于标准 Web 服务器。

### 4.3 软件接口

#### 4.3.1 后端 API 接口规范

| 规范项 | 内容 |
|:---|:---|
| 接口风格 | RESTful API |
| 数据格式 | JSON（请求与响应） |
| 认证方式 | JWT Token（放置在 Authorization Header） |
| 统一响应格式 | `{ code: 状态码, message: "提示信息", data: {} }` |
| 跨域配置 | CORS 跨域资源共享 |

#### 4.3.2 数据库接口

| 规范项 | 内容 |
|:---|:---|
| 数据库 | MySQL 8.x |
| ORM 框架 | MyBatis-Plus / JPA |
| 连接池 | HikariCP |

#### 4.3.3 缓存接口

| 规范项 | 内容 |
|:---|:---|
| 缓存服务 | Redis 6.x+ |
| 缓存内容 | 用户会话、验证码、热点商品数据 |
| 数据淘汰策略 | LRU（最近最少使用） |

#### 4.3.4 文件存储接口

| 规范项 | 内容 |
|:---|:---|
| 存储方式 | 本地文件存储或云 OSS |
| 图片处理 | 支持缩略图生成 |
| 访问方式 | HTTP/HTTPS URL |

### 4.4 通信接口

| 规范项 | 内容 |
|:---|:---|
| 前后端通信 | HTTP/HTTPS 协议 |
| WebSocket | 用于实时消息通知（可选） |
| API 版本控制 | URL 路径前缀 `/api/v1/` |

## 6. 数据需求 (Data Requirements)

### 6.1 核心数据实体

| 实体 | 描述 | 主要属性 |
|:---|:---|:---|
| **User** | 用户账户信息 | id, username, email, phone, password_hash, avatar, role, credit_score, status, created_at |
| **Address** | 收货地址 | id, user_id, receiver, phone, province, city, district, detail, is_default |
| **Category** | 商品分类 | id, name, parent_id, level, sort_order |
| **Product** | 商品信息 | id, user_id, title, description, price, condition, category_id, status, view_count, favorite_count, created_at |
| **ProductImage** | 商品图片 | id, product_id, url, sort_order |
| **Favorite** | 收藏记录 | id, user_id, product_id, created_at |
| **Order** | 订单信息 | id, order_no, buyer_id, seller_id, product_id, total_price, status, payment_method, logistics_company, logistics_no, created_at |
| **OrderLog** | 订单日志 | id, order_id, from_status, to_status, operator, remark, created_at |
| **Message** | 站内消息 | id, from_user_id, to_user_id, content, product_id, is_read, created_at |
| **Notification** | 系统通知 | id, user_id, title, content, type, is_read, created_at |
| **Review** | 交易评价 | id, order_id, from_user_id, to_user_id, rating, content, created_at |
| **Announcement** | 系统公告 | id, admin_id, title, content, status, created_at |

### 6.2 数据关系概览

```
User (1) ──→ (N) Product          // 用户发布商品
User (1) ──→ (N) Address          // 用户管理地址
User (1) ──→ (N) Order (buyer)    // 用户购买
User (1) ──→ (N) Order (seller)   // 用户出售
Product (1) ──→ (N) ProductImage  // 商品包含图片
Product (1) ──→ (N) Favorite      // 商品被收藏
Order (1) ──→ (N) OrderLog        // 订单状态变更记录
```

### 6.3 Redis 缓存策略

| 缓存对象 | Key 设计 | 缓存时间 | 说明 |
|:---|:---|:---|:---|
| 用户会话 Token | `session:{userId}` | 7 天 | 用于 JWT Token 刷新 |
| 验证码 | `captcha:{email/phone}` | 5 分钟 | 注册/找回密码验证码 |
| 热点商品信息 | `hot:product:{id}` | 1 小时 | 高访问量商品缓存 |
| 商品分类列表 | `category:tree` | 24 小时 | 分类树结构缓存 |
| 首页推荐商品 | `index:recommend` | 10 分钟 | 首页推荐列表缓存 |

## 7. 需求验证与确认 (Requirements Verification and Validation)

### 7.1 需求验证方法

| 验证方法 | 适用范围 |
|:---|:---|
| **需求评审** | 全体需求条目，由项目组成员和指导教师共同评审 |
| **原型验证** | 用户界面需求，通过 Vue3 原型页面进行验证 |
| **测试用例推导** | 每项功能需求至少对应一个测试用例 |
| **性能测试** | 非功能需求中的性能指标需通过压测工具验证 |

### 7.2 需求优先级与实现阶段

| 优先级 | 定义 | 对应需求 | 实施阶段 |
|:---|:---|:---|:---|
| **P0（必备）** | 系统核心功能，缺少则系统不可用 | 用户登录注册、商品 CRUD、订单生成与流转 | 第一阶段 |
| **P1（重要）** | 重要功能，影响用户体验 | 搜索筛选、图片上传、购物车、消息沟通 | 第二阶段 |
| **P2（增强）** | 增强功能，提升系统完善度 | 数据统计、后台管理、信誉评价 | 第三阶段 |
| **P3（可选）** | 扩展功能，锦上添花 | 推荐算法、敏感词过滤、纠纷处理 | 第四阶段 |

### 7.3 需求跟踪

需求跟踪矩阵（RTM）用于确保每项需求在后续的设计、实现和测试阶段均有对应的产出物。建议在项目实施过程中持续维护 RTM 表格。
