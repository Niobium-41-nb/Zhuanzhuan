\section{引言}

\subsection{目的}

本软件需求规格说明书（SRS）的目的是全面、系统地描述"校园二手物品交易平台"的功能需求、非功能需求、外部接口需求和约束条件，为后续的系统设计、编码实现、测试验收提供统一的依据和基准。

\subsection{范围}

本项目是一个基于 \textbf{Vue3 + SpringBoot + MySQL + Redis} 技术栈的校园二手物品交易平台综合实训项目，旨在实现一个功能完善、界面友好、安全可靠的校园 C2C 二手交易 Web 应用。

本系统覆盖以下功能域：
\begin{itemize}[itemsep=3pt]
    \item 用户管理
    \item 商品管理
    \item 订单交易
    \item 消息与沟通
    \item 后台管理
    \item 系统安全与审计
\end{itemize}

\section{总体描述}

\subsection{产品视角}

本系统是一个独立的校园二手物品交易 Web 应用平台，采用前后端分离架构。系统不依赖其他外部系统运行，但通过与数据库（MySQL）和缓存（Redis）的交互实现数据持久化与高性能访问。

\textbf{系统上下文：}
\begin{itemize}[itemsep=3pt]
    \item \textbf{前端：} Vue3 单页应用（SPA），运行于用户浏览器
    \item \textbf{后端：} SpringBoot RESTful API 服务
    \item \textbf{数据库：} MySQL 关系型数据库
    \item \textbf{缓存层：} Redis 缓存服务
    \item \textbf{文件存储：} 本地存储或云 OSS
\end{itemize}

\subsubsection{密码找回}

\textbf{标识：} FR-UM-03

系统应支持用户通过绑定的邮箱/手机号重置密码：
\begin{enumerate}[itemsep=3pt]
    \item 用户提交邮箱/手机号
    \item 发送重置链接/验证码
    \item 验证通过后允许设置新密码
    \item 新密码加密存储并清除旧 Token
\end{enumerate}

\subsubsection{个人信息管理}

\textbf{标识：} FR-UM-04

已登录用户可在个人中心查看和编辑个人信息，包括：头像上传（jpg/png，限制2MB）、昵称修改、个人简介编辑、绑定邮箱/手机号更换。前置条件为用户已登录（JWT 校验通过）。

\subsubsection{收货地址管理}

\textbf{标识：} FR-UM-05

用户可添加、编辑、删除、设置默认收货地址。地址字段包括：收件人、联系电话、省份/城市/区县、详细地址、邮政编码。

\subsubsection{用户信誉评分}

\textbf{标识：} FR-UM-06

交易完成后买卖双方可互相评分（1--5星），系统汇总显示用户信誉等级。

\subsubsection{商品浏览与搜索}

\textbf{标识：} FR-GM-02

用户可浏览商品列表，支持分类筛选、关键词搜索和多维度排序：
\begin{itemize}[itemsep=3pt]

\subsubsection{商品状态管理}

\textbf{标识：} FR-GM-04

卖家可管理自己发布的商品状态。状态流转：待审核 $\rightarrow$ 在售（审核通过）$\rightarrow$ 已下架 / 已售出 / 审核驳回。支持上架、下架、编辑（仅下架状态可编辑）、删除操作。

管理员可审核卖家发布的商品，包括待审核列表、商品详情预览、审核通过/驳回（填写驳回原因）。

\textbf{标识：} FR-AM-05

\subsection{可选扩展功能}

\begin{itemize}[itemsep=5pt]

\end{itemize}

    \item 交互方式：鼠标点击、键盘输入、页面路由跳转
\end{itemize}

\subsubsection{后端 API 接口规范}

\end{itemize}

\subsection{通信接口}

\begin{itemize}[itemsep=3pt]

    \item WebSocket：用于实时消息通知（可选）
    \item API 版本控制：URL 路径前缀 \texttt{/api/v1/}
\end{itemize}
