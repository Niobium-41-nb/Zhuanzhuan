## 2. 接口规范

### 2.1 通用规范

| 项目 | 规范 |
|:---|:---|
| **接口风格** | RESTful API |
| **基础路径** | `/api/v1/` |
| **数据格式** | JSON（UTF-8 编码） |
| **字符编码** | UTF-8 |
| **认证方式** | JWT Token（Bearer 认证） |
| **内容类型** | `application/json`（上传接口使用 `multipart/form-data`） |

### 2.2 通用请求头

| 请求头 | 说明 | 必填 |
|:---|:---|:---:|
| `Content-Type` | 内容类型：`application/json` | ✔ |
| `Authorization` | JWT Token：`Bearer <token>` | 需认证接口 |
| `Accept-Language` | 语言偏好：`zh-CN`（默认） | ✘ |

### 2.3 统一响应格式

#### 2.3.1 成功响应

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

#### 2.3.2 失败响应

```json
{
  "code": 400,
  "message": "错误描述信息",
  "data": null
}
```

#### 2.3.3 分页响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "size": 10,
    "pages": 10
  }
}
```

### 2.4 全局状态码

| 状态码 | 说明 |
|:---:|:---|
| **200** | 请求成功 |
| **201** | 创建成功 |
| **400** | 请求参数错误 |
| **401** | 未认证（Token 无效或过期） |
| **403** | 无权限访问 |
| **404** | 资源不存在 |
| **405** | 请求方法不允许 |
| **409** | 资源冲突（如重复注册） |
| **415** | 不支持的媒体类型 |
| **422** | 请求参数校验失败 |
| **429** | 请求过于频繁（接口防刷） |
| **500** | 服务器内部错误 |

### 2.5 业务状态码

| 业务码 | 说明 |
|:---:|:---|
| `1001` | 账户已被禁用 |
| `1002` | 密码错误次数过多，账户已锁定 |
| `1003` | 验证码错误或已过期 |
| `1004` | Token 已过期，请刷新 |
| `2001` | 商品已售出或已下架 |
| `2002` | 商品审核未通过 |
| `2003` | 超出每日发布限额 |
| `3001` | 订单状态不允许当前操作 |
| `3002` | 订单超时未支付已自动取消 |
| `4001` | 消息发送过于频繁 |
| `5001` | 文件上传超出大小限制 |
| `5002` | 不支持的文件类型 |

### 2.6 认证方式

所有需要登录认证的接口，请求头中必须携带有效的 JWT Token：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

- **Access Token**：有效期 **2 小时**，用于正常接口访问
- **Refresh Token**：有效期 **7 天**，用于刷新 Access Token

**认证流程：**

```
登录 ──→ 获取 Access Token + Refresh Token
              │
        请求携带 Token
              │
              ▼
    ┌─── Token 有效 ──→ 正常处理请求
    │
    └─── Token 过期 ──→ 返回 401
              │
        使用 Refresh Token 刷新
              │
              ▼
    ┌─── Refresh Token 有效 ──→ 返回新 Token
    │
    └─── Refresh Token 过期 ──→ 重新登录
```

### 2.7 通用分页参数

| 参数名 | 类型 | 默认值 | 说明 |
|:---|:---:|:---:|:---|
| `page` | Integer | 1 | 页码，从 1 开始 |
| `size` | Integer | 10 | 每页条数，最大 100 |
| `sort` | String | — | 排序字段，如 `created_at` |
| `order` | String | `desc` | 排序方向：`asc` / `desc` |

### 2.8 通用查询参数

对所有列表接口，支持以下通用查询参数：

| 参数名 | 类型 | 说明 |
|:---|:---:|:---|
| `keyword` | String | 关键词模糊搜索 |
| `status` | String | 按状态筛选 |
| `startDate` | String | 开始时间（yyyy-MM-dd） |
| `endDate` | String | 结束时间（yyyy-MM-dd） |

## 3. 用户模块接口

### 3.1 用户注册

**接口说明：** 新用户注册，支持邮箱或手机号注册。

- **URL：** `/api/v1/user/register`
- **方法：** `POST`
- **权限：** 公开

**请求参数：**

```json
{
  "username": "zhangsan",
  "password": "Abc12345",
  "email": "zhangsan@edu.cn",
  "phone": "13800138000",
  "code": "123456"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|:---|:---:|:---:|:---|
| `username` | String | ✔ | 用户名，2-20位字母数字下划线 |
| `password` | String | ✔ | 密码，8-20位，包含字母和数字 |
| `email` | String | * | 邮箱（email/phone 至少填一个） |
| `phone` | String | * | 手机号 |
| `code` | String | ✔ | 验证码 |

**成功响应 (201)：**

```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "userId": 1,
    "username": "zhangsan"
  }
}
```

**错误响应：**

```json
{ "code": 409, "message": "用户名已存在", "data": null }
{ "code": 422, "message": "密码长度不能小于8位", "data": null }
```

```json
  }

```
