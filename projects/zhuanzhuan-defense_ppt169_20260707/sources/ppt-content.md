## 阮崇轩 — 前端开发

### 前端工程搭建
- Vue3 + TypeScript + Vite 项目初始化
- 配置 Element Plus 按需导入、Pinia 状态管理、Vue Router、Axios
- 设计 CSS Variables 设计令牌体系（主色森林绿 #2D6A4F、辅色暖橙 #E76F51）
- 引入 Google Fonts 品牌字体，全局覆盖 Element Plus 默认样式

### 用户认证模块
- 登录页：渐变背景 + 浮动几何图形动画，支持用户名/邮箱/手机号登录
- 注册页：邮箱验证码注册，60 秒倒计时防重复点击
- 找回密码页：两步流程设计（邮箱验证 → 设置新密码），密码强度校验
- Axios 封装：Token 自动注入、401 自动刷新（请求队列排队机制）、403 自动清除过期登录
- 修复密码重置验证码被提前消耗的 Bug

### 消息通知系统
- 消息页完整重写：双栏布局（会话列表 + 聊天面板），聊天气泡设计，Enter 快捷发送
- URL 参数自动打开会话（?to=userId），支持发起新对话
- 30 秒轮询刷新未读消息数和通知数
- 头部新增通知铃铛图标 + Popover 下拉面板，点击标记已读
- 后端消息/订单状态自动创建系统通知

### 订单详情页
- 买家-卖家双视角操作判断
- 买家操作：支付、取消、确认收货、评价（星级 + 文字）
- 卖家操作：录入物流发货（选择物流公司 + 输入单号）
- 订单状态时间线可视化

### UI 动效优化
- 页面过渡动画（fade/slide）、回到顶部按钮（弹性缩放动画）
- 骨架屏加载（Shimmer 动画）、商品卡片交错入场
- 导航栏滚动感知（毛玻璃模糊 + 阴影）
- 全局按钮微交互（hover 上浮、active 回弹）

## 刘轩霆 — 后端开发 A

### 用户认证模块
- Spring Security + JWT 无状态认证架构
- 实现注册/登录/Token刷新/登出接口
- JWT Token 黑名单机制（Redis）
- 登录频率限制（10次/15分钟）、验证码发送频率限制（3次/5分钟）
- BCrypt 密码加密
- 邮箱验证码发送（JavaMailSender）

### 商品模块
- 商品发布（多图关联、悲观锁防双重出售）
- 商品列表查询（分页 + 多条件筛选 + 排序 + 全文搜索）
- 商品详情、编辑、删除、状态管理
- 批量加载优化（分类名、卖家信息，避免 N+1 查询）
- 浏览量统计（非卖家浏览才计数）

### 安全框架
- JwtAuthenticationFilter：Token 验证、黑名单检查、用户状态检查
- SecurityConfig：公开/认证/管理员三级权限控制
- CORS 跨域配置
- 全局异常处理

### 订单核心流程
- 订单创建（悲观锁 + 自买自卖校验）
- 订单状态流转：待付款 → 待发货 → 待收货 → 已完成 / 已取消
- 订单日志记录（状态变更历史）
- 商品售出状态同步

## 技术架构

- 前端：Vue3 + TypeScript + Vite + Element Plus + Pinia + Axios + ECharts
- 后端：SpringBoot 3.2 + MyBatis-Plus 3.5 + Spring Security + JWT + Knife4j
- 数据库：MySQL 8.0（InnoDB、全文索引）
- 缓存：Redis 7（验证码、Token 黑名单、分类树缓存）
- 部署：Docker + Docker Compose（4 容器编排）
- 安全：BCrypt 密码加密、JWT 认证、CORS 跨域、RBAC 权限

## 项目成果

- 17 个用户账号、217 件商品数据
- 30+ 后端 API 接口
- 21 个前端页面（含 5 个管理后台页面）
- 7 份项目文档（SRS、架构设计、数据库设计、接口设计、测试报告、用户手册、工作日报）
- 完整的 Docker 一键部署方案

## 技术亮点

- JWT 双 Token 机制（Access Token 2h + Refresh Token 7d）+ 自动无感刷新
- Redis 验证码隔离（类型前缀防跨用途滥用）+ 频率限制防暴力破解
- 订单悲观锁防双重出售 + 商品售出状态同步
- N+1 查询批量优化（分类名/卖家信息一次加载）
- CSS Variables 设计令牌体系 + Element Plus 全局主题覆盖
- 骨架屏加载 + 页面过渡动画 + 滚动感知导航栏
